Asyncronous Programming

                

                    
Allows multiple things to happen at once

                    
sendTwitterUpdate("I just learned Asyncronous Programming!", function(response){
    console.log("Twitter says:" + response);
});

sendFacebookUpdate("I just learned Asyncronous Programming!" function(response) {
    console.log("Facebook says:" + response);
});
                    


                

                

            


            

                
Asyncronous Programming

                

                    
Multitasking is something computers are good at - using spare miliseconds to do things.

                    
It's something programmers are not always so great at - race conditions can be a problem.

                

                
Demonstrate what might happen if I counted on twitter being done before facebook, etc.

            


            

                
Non-blocking I/O

                

                    
The reason we can do this is because of non-blocking I/O. We can start a disk read, or some internet activity, and while we're waiting for that to finish, we can do something else.

                

                

            


            

                
Events

                

                    
Responding to events is how we program in node.js. We might respond to a Request event, or a file being loaded into memory, or a timed event.

                    
Running things as the result of an event is how we can manage the flow of control in a node.js web application.
